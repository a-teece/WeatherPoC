## Contradictions

### C1: The 100% coverage gate is stated flatly upstream but carries an exclusion downstream
**Sources:** `Technical-Context.MD` (Overriding Principle 5) vs. `PRD.md` (Testing Decisions) / `Roadmap.md` (Feature 1)
**A claims:** "**100% unit-test coverage.** Coverage is a hard gate, not an aspiration. New production code arrives with the unit tests that cover it." ÔÇö `Technical-Context.MD`
**B claims:** "**Excluded from unit tests** (coverage-excluded as untestable UI/wiring): the XAML Views, the DI composition root, and Serilog bootstrap." ÔÇö `PRD.md`; and "the coverage gate passes vacuously ÔÇŽ The XAML View, DI composition root, and Serilog bootstrap are coverage-excluded as untestable UI/wiring" ÔÇö `Roadmap.md` Feature 1.
**Likely authority:** A (`Technical-Context.MD` outranks `PRD.md` and `Roadmap.md` in the artefact hierarchy). The flat "100%, hard gate" is the higher-authority wording, but the team's clear and repeated intent (in both downstream artefacts) is to exclude UI/wiring via coverage attributes.
**Why this matters:** An AFK agent reading only Principle 5 could either (a) fail a build that legitimately excludes the XAML View, or (b) waste effort trying to unit-test untestable XAML/DI/bootstrap. This is the classic constraint-vs-derived drift that becomes a Design Failure.
**Suggested resolution:** Update `Technical-Context.MD` Principle 5 to state the UI/wiring coverage-exclusion explicitly (e.g. "100% of non-UI/wiring production code; XAML Views, DI composition root, and Serilog bootstrap are coverage-excluded"). Don't relax the gate ÔÇö make the exclusion authoritative at the point where the gate is defined, so the highest-authority artefact and the derived ones agree.

## Vocabulary drift

### V1: Term "Place Search" referred to as "manual search" / "manual entry" / "manual prompt"
**Defined in `Context.MD` as:** "**Place Search** ÔÇö Finding a Location by its human-readable place name so the User can choose one explicitly. ÔÇŽ the only way to obtain an Active Location when automatic detection fails."
**Used in `docs/adr/0001-location-detection-cascade.md` as:** title "Location detection cascades GPS Ôćĺ IP geolocation Ôćĺ **manual search**"; considered options "**GPS Ôćĺ manual search** (no IP service)"; consequence "drops the User straight into a **manual prompt** on first run."
**Also used in `PRD.md` (Story 4) as:** "ÔÇŽvia Place Search only when automatic detection fails entirely, so that **manual entry** is the last resort."
**Assessment:** Accidental synonyms for the same glossary term, not a new concept. The cascade's terminal step *is* Place Search per the glossary.
**Suggested resolution:** Rename "manual search" / "manual prompt" / "manual entry" to "Place Search" in ADR 0001 and PRD Story 4. (Alternatively, add a one-line alias note to the `Context.MD` "Place Search" entry ÔÇö but renaming the downstream usages is cleaner, since the glossary term already exists.)

### V2: "underlying value" (Context.MD) vs "canonical value" (PRD / Roadmap)
**Defined in `Context.MD` as:** "**Measurement Unit** ÔÇö A display concern only: changing it changes how a value is shown, never the **underlying value**."
**Used in `PRD.md` (Implementation Decisions, Units) as:** "converts a raw **canonical value** into the User's chosen Measurement Unit ÔÇŽ never mutates the **underlying value**"; and `Roadmap.md` Feature 4: "never re-fetching or mutating the underlying **canonical value**."
**Assessment:** Same concept ÔÇö the unit-independent stored reading. Low priority; this reads as elaboration rather than a true divergence.
**Suggested resolution (optional):** If "canonical value" is going to be load-bearing in the formatter Spec, name it in `Context.MD` (e.g. under Weather Variable or Measurement Unit) so it's defined once. Otherwise standardise downstream on "underlying value."

WMO codes and the icon/label rendering are the implementation behind the defined **Weather Condition**; `Context.MD` is explicitly "glossary only ÔÇö no implementation details," so their absence from it is by design, not drift.
